# 用户指南

## 今日机会

从 `/dashboard` 开始。机会有 P0-P3 优先级，所有分数都应能追溯到 Signal 和 Evidence。没有真实数据时不会展示示例公司。

## 导入旧数据

进入 `/dashboard/v1-migration`：下载原始备份、选择备份、预检每条项目、人工处理冲突，再执行导入。回滚只软删除本次创建的记录；原始备份和 SourceRun 会保留。

## 数据来源

在 `/dashboard/sources` 可手工添加公开 URL Evidence。Serper 需要在本地配置服务端密钥；Customs CSV 只导入用户合法取得的文件。三者都不会自动创建客户或机会。

## 核实与优先级

P0 表示已核实公司、近期强信号、可信日期与高质量证据都齐全。P1/P2 仍需要人工研究；P3 表示同行、过期、目录/商品页单一来源或主体不明。电商商品页不能单独视为采购需求。

## 开发动作

在 `/dashboard/actions` 记录任务、联系尝试和草稿。草稿仅供人工复制和审核，系统绝不会自动发送邮件、LinkedIn 或 WhatsApp 信息。
